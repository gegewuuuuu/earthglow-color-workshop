## Host（海外）

```
https://grsaiapi.com
```

## Host（国内直连）

```
https://grsai.dakka.com.cn
```

## 拼接方式

Host + 接口路径，例如：
`https://grsai.dakka.com.cn/v1/draw/nano-banana`

# 2）兼容 Gemini 官方接口格式（可选）

支持 Gemini 官方的接口风格，只需要：

- **基础地址换成 Grsai 的地址**
- **模型名**把 `gemini-2.5-flash-image` 改成 `nano-banana-fast`

示例 URL：
`https://grsai.dakka.com.cn/v1beta/models/nano-banana-fast:streamGenerateContent`

# 3）绘图接口（生成图片）

## 接口信息

- **接口名**：Nano Banana 绘画接口
- **Method**：`POST`
- **Path**：`/v1/draw/nano-banana`
- **返回方式**：支持 `stream` 或 `webhook 回调`

------

## 请求 Headers

```http
Content-Type: application/json
Authorization: Bearer apikey
```

------

## 请求参数（JSON）

### 示例

```json
{
  "model": "nano-banana-fast",
  "prompt": "提示词",
  "aspectRatio": "auto",
  "imageSize": "1K",
  "urls": [
    "https://example.com/example.png"
  ],
  "webHook": "https://example.com/callback",
  "shutProgress": false
}
```

------

## 参数说明

### 1）model（必填 | string）

示例：`"nano-banana-fast"`

支持模型（截图里列出的）：

- nano-banana-fast
- nano-banana
- nano-banana-pro
- nano-banana-pro-vt
- nano-banana-pro-cl
- nano-banana-pro-vip
- nano-banana-pro-4k-vip

------

### 2）prompt（必填 | string）

提示词，例如：
`"一只可爱的猫咪在草地上玩耍"`

------

### 3）urls（可选 | array）

参考图输入（支持 URL 或 Base64）

示例：

```json
"urls": ["https://example.com/example.png"]
```

------

### 4）aspectRatio（可选 | string）

输出图片比例，默认 `auto`

支持：

- auto
- 1:1
- 16:9
- 9:16
- 4:3
- 3:4
- 3:2
- 2:3
- 5:4
- 4:5
- 21:9

------

### 5）imageSize（可选 | string）

示例：`"1K"`
输出图大小支持：

- 1K（默认）
- 2K
- 4K

注意：分辨率越高，生成时间越长。

另外截图里写了**部分模型限制**：

- nano-banana-pro / pro-vt / pro-cl：支持 1K/2K/4K（文档表示支持）
- nano-banana-pro-vip：只支持 1K、2K
- nano-banana-pro-4k-vip：只支持 4K

------

### 6）webHook（可选 | string）

回调地址，例如：
`https://your-webhook-url.com/callback`

- 开启后：任务进度与结果会通过 `POST` 回调给你
- 回调请求头：`Content-Type: application/json`

**如果你不想用回调，而是想轮询拿结果：**

- `webHook` 传 `"-1"`
- 接口会立即返回一个 `id`，你再用 `/v1/draw/result` 去查结果

------

### 7）shutProgress（可选 | boolean）

示例：`false`

含义：关闭进度回复，直接只回最终结果（文档建议配合 webhook 使用）
默认：`false`

------

# 4）webHook 回调返回格式

## 4.1 提交任务后（返回 id）

当你请求里填写了 webHook，接口会先返回一个 id（用于对应回调数据）：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "id": "id"
  }
}
```

- `code = 0` 表示成功
- `data.id`：任务 id（后面回调 / 查询都用它）

------

## 4.2 webHook 实际回调内容（平台 POST 给你）

```json
{
  "id": "xxxxx",
  "results": [
    {
      "url": "https://example.com/example.png",
      "content": "这是一只可爱的猫咪在草地上玩耍"
    }
  ],
  "progress": 100,
  "status": "succeeded",
  "failure_reason": "",
  "error": ""
}
```

字段解释：

- `id`：任务 id
- `results`：生成结果数组
  - `url`：图片链接（文档注明有效期约 2 小时）
  - `content`：提示词内容（回传）
- `progress`：进度 0~100
- `status`：
  - `running`：进行中
  - `succeeded`：成功
  - `failed`：失败
- `failure_reason`：
  - `output_moderation`：输出违规
  - `input_moderation`：输入违规
  - `error`：其他错误
- `error`：错误详情，例如 `"Invalid input parameters"`

# 5）轮询查询结果接口（手动获取结果）

> 适用于你 **webHook = "-1"** 的情况，或者你就是想自己查。

## 接口信息

- **Method**：`POST`
- **Path**：`/v1/draw/result`

------

## 请求参数

```json
{
  "id": "xxxxx"
}
```

------

## 响应示例

```json
{
  "code": 0,
  "data": {
    "id": "xxxxx",
    "results": [
      {
        "url": "https://example.com/example.png",
        "content": "这是一只可爱的猫咪在草地上玩耍"
      }
    ],
    "progress": 100,
    "status": "succeeded",
    "failure_reason": "",
    "error": ""
  },
  "msg": "success"
}
```

响应字段：

- `code`：
  - `0` 成功
  - `22` 任务不存在（截图里写的）
- `msg`：状态信息
- `data`：生成结果（结构同 webhook 回调）

# 6）最简调用示例（可直接改）

## 6.1 走 webhook 回调（推荐）

```bash
curl -X POST "https://grsai.dakka.com.cn/v1/draw/nano-banana" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model":"nano-banana-fast",
    "prompt":"一只可爱的猫咪在草地上玩耍",
    "aspectRatio":"1:1",
    "imageSize":"1K",
    "webHook":"https://your-domain.com/callback",
    "shutProgress":false
  }'
```

## 6.2 走轮询（webHook = "-1"）

提交任务：

```bash
curl -X POST "https://grsai.dakka.com.cn/v1/draw/nano-banana" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model":"nano-banana-fast",
    "prompt":"一只可爱的猫咪在草地上玩耍",
    "webHook":"-1"
  }'
```

然后用返回的 `id` 查结果：

```bash
curl -X POST "https://grsai.dakka.com.cn/v1/draw/result" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "id":"返回的任务id"
  }'
```
