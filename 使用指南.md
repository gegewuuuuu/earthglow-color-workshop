# EarthGlow 使用指南

## 快速启动

### 方式一：使用 Python HTTP 服务器（推荐）

```bash
# 在项目目录下运行
python -m http.server 8000
```

然后在浏览器打开：`http://localhost:8000/color1.html`

### 方式二：使用 Node.js http-server

```bash
# 安装 http-server（如果没有）
npm install -g http-server

# 启动服务器
http-server -p 8000
```

### 方式三：直接双击打开（可能有CORS限制）

直接双击 `color1.html` 文件，但某些功能可能受限。

## 功能说明

### 1. 文本生成图片

**快速开始：**
1. 点击右侧面板的预设按钮：
   - "Golden Hours" - 生成金色时光风格
   - "80s Retro Disco" - 生成80年代复古迪斯科风格
   - "Morandi's Afternoon" - 生成莫兰迪下午风格

2. 或者在输入框输入自定义提示词：
   - "Sunset over ocean with warm colors"
   - "Vibrant tropical beach"
   - "Dreamy purple night sky"
   - "Colorful autumn forest"

3. 按下 Enter 键或点击发送按钮

4. 等待 10-30 秒，图片生成完成后会：
   - 显示在主画布区域
   - 自动提取 5 种主要颜色显示在左侧色点
   - 保存到历史记录

### 2. 颜色提取

生成图片后，系统会自动：
- 分析图片中的主要颜色
- 使用 k-means 聚类算法提取 5 种主导色
- 在左侧 "Original artwork" 面板下方显示色点
- 鼠标悬停色点可查看颜色代码

### 3. 历史记录

**查看历史：**
- 所有生成的图片会显示在底部 "History" 面板
- 最多保存 20 条记录
- 点击缩略图可恢复之前的生成结果

**历史功能：**
- 自动保存到浏览器 localStorage
- 刷新页面后历史记录依然存在
- 每条记录包含：图片、提示词、颜色、时间戳

### 4. 下载图片

1. 生成图片后，点击右侧下方的下载按钮
2. 图片会以 PNG 格式下载，文件名为 `earthglow_时间戳.png`

## API 配置

**当前配置（config.js）：**
```javascript
{
  apiKey: 'your-api-key',
  baseURL: 'https://grsai.dakka.com.cn',
  model: 'nano-banana-fast',
  aspectRatio: '1:1',
  imageSize: '1K'
}
```

**如需修改：**
编辑 `config.js` 文件修改以下参数：
- `apiKey` - 你的 GRSAI API 密钥
- `model` - 模型选择（nano-banana-fast/nano-banana/nano-banana-pro）
- `imageSize` - 图片大小（1K/2K/4K）
- `aspectRatio` - 长宽比（1:1/16:9/9:16/4:3等）

## 技术细节

### 使用的技术栈
- **前端框架：** 纯 Vanilla JavaScript（无框架依赖）
- **API：** GRSAI Nano Banana 图片生成 API
- **颜色提取：** HTML5 Canvas + K-Means 聚类算法
- **数据存储：** localStorage（浏览器本地存储）
- **样式：** 玻璃态设计（Glassmorphism）

### 文件结构
```
web/
├── color1.html          # 主页面
├── config.js            # API配置
├── js/
│   ├── api-client.js    # API客户端
│   ├── ui-controller.js # UI控制器
│   ├── color-extractor.js # 颜色提取器
│   ├── storage-manager.js # 存储管理器
│   └── utils.js         # 工具函数
└── Assets/              # 图片资源
```

### 核心模块说明

**1. GRSAIClient（API客户端）**
- 负责与 GRSAI API 通信
- 轮询机制获取生成结果
- 错误处理和重试逻辑

**2. ColorExtractor（颜色提取器）**
- 使用 Canvas API 采样像素
- K-means 聚类算法（k=5）
- 过滤透明和极端亮度像素

**3. StorageManager（存储管理器）**
- localStorage 读写操作
- 自动清理超出限制的历史
- 数据导入导出功能

**4. UIController（UI控制器）**
- 协调所有模块
- 处理用户交互
- 管理加载状态和错误提示

## 常见问题

### Q1: 生成时间太长怎么办？
**A:** 正常生成时间为 10-30 秒。如果超过 60 秒会自动超时。可以：
- 使用 `nano-banana-fast` 模型（最快）
- 选择较小的图片尺寸（1K）
- 检查网络连接

### Q2: 提示 API 错误？
**A:** 可能的原因：
- API 密钥无效或过期（检查 config.js）
- 网络连接问题
- API 服务暂时不可用
- 超出使用配额

### Q3: 历史记录丢失？
**A:** localStorage 数据可能被清理，原因：
- 浏览器清理了缓存/Cookies
- 使用了隐私/无痕模式
- 超出存储配额（5MB限制）

解决方案：使用导出功能备份历史。

### Q4: 颜色提取不准确？
**A:** 颜色提取使用 k-means 算法，可能的原因：
- 图片主要颜色分布不均
- 背景色干扰
- 算法采样率设置

### Q5: 无法下载图片？
**A:** 可能是浏览器安全限制：
- 检查弹窗拦截设置
- 允许文件下载权限
- 使用支持的现代浏览器

### Q6: 跨域（CORS）错误？
**A:** 必须使用 HTTP 服务器运行，不能直接打开 HTML 文件：
```bash
python -m http.server 8000
```

## 限制说明

1. **API 密钥暴露：**
   - 当前配置中 API 密钥在浏览器中可见
   - 仅适用于个人使用或开发环境
   - 生产环境建议使用后端代理

2. **存储限制：**
   - localStorage 容量约 5-10MB
   - 最多保存 20 条历史记录
   - 图片以 data URI 形式存储

3. **图片 URL 有效期：**
   - GRSAI 返回的图片 URL 约 2 小时后失效
   - 已通过转换为 data URI 解决

4. **生成限制：**
   - 单次只能生成一张图片
   - 不支持批量生成
   - 不支持上传参考图（Phase 2 功能）
